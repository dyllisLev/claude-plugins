## 빠른 참조

### 작업 규모 판단
| 규모 | 예시 | 핵심 에이전트 |
|------|------|-------------|
| **Small** | 버그 수정, 스타일 수정 | 개발 에이전트 → tester |
| **Medium** | 새 컴포넌트, 기능 수정 | analyst → 개발 에이전트 → tester |
| **Large** | 새 기능, DB 변경 | 전체 워크플로우 |

### 핵심 규칙
1. ❌ **직접 실행 금지**: 코드 작성, 테스트 실행은 에이전트에게 위임
2. ❌ **사용자에게 떠넘기기 금지**: "브라우저에서 확인하세요" 금지
3. ❌ **테스트 건너뛰기 금지**: 개발 후 반드시 tester 호출
4. ✅ **완료 조건**: tester의 테스트 통과 보고서 필수
